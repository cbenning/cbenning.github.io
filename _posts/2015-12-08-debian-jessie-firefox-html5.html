--- layout: post title: Making HTML5 work with Firefox on Debian Jessie categories: [] tags: [Linux,Debian,HTML5,Firefox] status: publish type: post published: true comments: true meta: _edit_last: '1' --- I was getting a bit annoyed at all the disjoint posts around the internet about making HTML5 work with my Firefox install on my work laptop so I thought I'd contribute to the noise. Getting started, if you go to http://youtube.com/html5 and you see something like the screenshot below, you might want to try out my instructions. ![HTML5 No worky]({{ site.url }}/assets/img/html5-before.png) You might see x264 also in red, this guide should fix that as well.

Install Debian multimedia repository

(As root) Add the repo to your sources: {% highlight bash %} echo "http://www.deb-multimedia.org jessie main non-free" >> /etc/apt/sources.list.d/deb-multimedia.list {% endhighlight %}
Update sources and install keyring and update sources again: {% highlight bash %} sudo apt-get update sudo apt-get install deb-multimedia-keyring sudo apt-get update {% endhighlight %}

Install Codecs

Remove any leftover ffmpeg implementation from the original repo: {% highlight bash %} sudo apt-get remove ffmpeg {% endhighlight %}
Install Packages: {% highlight bash %} sudo apt-get install ffmpeg x264 gstreamer0.10-ffmpeg {% endhighlight %}

Enable Mediasources in Firefox

Open firefox, go to 'about:config' in your addressbar. Search for 'mediasource'.
Set the following two keys to 'true': 
media.mediasource.enabled
media.mediasource.webm.enabled

Restart Firefox

Validate Changes

Go back to http://youtube.com/html5 in firefox and hopefully you will see more blue: ![HTML5 Worky]({{ site.url }}/assets/img/html5-after.png)

Profit!

See! Linux users can have nice things too.
