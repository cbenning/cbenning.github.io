--- layout: post title: Google Hangouts plugin crashes in Debian Testing 64-bit categories: [] tags: [Linux,Debian,Google,Hangouts] status: publish type: post published: true meta: _edit_last: '1' --- After an update the other day, Google Hangouts stopped working in Firefox and in Chrome. The symptoms were a collection of items, including the browser not registering that the plugin was installed, a prompt saying the plugin has crashed and a message in dmesg about the plugin segfaulting. It took me a while to find the correct search term and log entry to end at the solution that solved it for me. I found the log file in '~/config/google'. After examining I found the following mixed in the messages: {% highlight bash %} cat ~/.config/google-googletalkplugin/gtbplugin.log ... ... Connection to GoogleTalkPlugin failed, reason=111 ... {% endhighlight %} After some searching, I ended up finding a thread in Google Support with a number of suggestions. Ultimately, I found that my update on Debian pulled down 'libudev1' when 'libudev0' was already installed. There was no conflict, but the plugin seemed to use 'libudev0' even though it required 'libudev1'. Simply uninstalling libudev0 was enough to do it for me, but if you need to keep it for some reason, one can force the plugin to use 'libudev1' with a rather-hacky symbolic link: {% highlight bash %} ln -s /lib/x86_64-linux-gnu/libudev.so.1 /opt/google/chrome/lib/libudev.so.0 {% endhighlight %} I hope this helps someone, as I found it took much longer than I'd like to find this solution, even though it was not that uncommon.
